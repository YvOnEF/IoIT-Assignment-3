Project Description

Overview

Our project is a mobile application that is essentially a virtual wallet designed to be a unified hub for all forms of virtual cards, QR codes and virtual tickets. This idea can be innovative because there are many different types of services that can store credit cards (eg: apple pay and google play), and some less popular virtual cards and/or QR Codes used in other non wallet apps may use a different application or not even be available on a virtual wallet service, meaning a user might need multiple applications and multiple accounts to use all their virtual cards on their phone. This application will be designed so that all these cards and scannable codes and items can be stored in one place, as a unified “eWallet”, making a card holder’s life easier and more efficient when dealing with scannable items on their phone.

Motivation

This idea has come from personal experiences, with statistics and global trends making this idea more viable. When I make cashless payments, the majority of those transactions are made through Apple pay which I find to be much more convenient than taking out my physical wallet. The only time I need to regularly take out my wallet is when taking public transport to touch on and off my miki, which for me personally is a mild inconvenience when we already have these technologies in place in our phones through near field communication (NFC). Although a miki can be stored on mobile through google pay, the initial inconvenience still stands, having multiple wallet/card applications can be less efficient and convenient than a simple physical wallet! Especially when we are moving into a more digitised society. The solution is simple, people can have all their physical cards stored in one wallet, so all that is needed is one application to store everything, not multiple applications. This idea of convenience and practicality is the key motivation for my project. Data proves that this type of technology is becoming increasingly popular, with virtual wallets becoming increasingly more adopted. With 900 million transactions at the start of the pandemic increasing to almost 1.5 billion in 2021, a 25% increase, which is set to go above 33% in the coming years. The growth has been so large in fact that 45% of all eCommerce transactions were through mobile in 2021, which is twice as large as the payments done via debit cards and three times of debit card payments.

Landscape

There are many similar products like Apple pay and Google pay in the market, however our product differs from them with these two features:

  • User Inserted Scannable Cards (locally stored): If a scannable item isn't supported by the institution, then the user can take a photo or screenshot of QR code or other scannable item that does not send a signal and then save it in the wallet, making it so that the code can still be scanned in the app without redirecting to another app or even referring to a physical voucher/docket that may have been given to them.
  • Direct Scan: With Apple, Google and Samsung announcing their phones can be used as terminals for businesses to accept payments through their NFC chips, our application has the potential to use the NFC chips in modern smartphones to read the signal provided by a card in close proximity to the phone and duplicate the signal to be stored on the application, to be later used through the application. This should work with all cards that use radio frequency identification (RFID) or have an NFC chip if functional. This should be ideal for cards that are used by hotels or businesses and education institutions that use these cards only in physical form

  • Sensitive information: Organizations need to ensure that they do not collect sensitive information about people, such as their religion, political beliefs or criminal records, without checking the applicable laws.
  • Collection of personal information: When an organization collects information, it should only collect the information it needs. The organization should inform people that their information is being collected.
  • Data quality: Ensure that the information collected is accurate, complete and up-to-date.
  • Data Security: Information must be protected from misuse, loss, authorized access, modification or disclosure. Reasonable steps must be taken to destroy or de-identify personal information that is no longer needed.
  • Transborder data flows: If your personal information travels outside Victoria, your privacy protections must travel with it.
  • Access and correction: When an organization collects information it should allow people to see the information it collects about them and provide them with the opportunity to correct it if it is inaccurate.

    Apple software:

    Swift

    Swift is a modern intuitive programming language for iOS, which is designed safely and produces software that runs quickly. Swift is based on the latest research on programming languages, with decades of combined experience of Apple platform development. Named parameters are expressed cleanly in a syntax that enables APIs in Swift to more easily be read and maintained. Programmers using Swift do not need to input semicolons, inferred typing makes code more presentable and less error prone while headers are eliminated by modules and namespaces are provided. Strings use a UTF-8 based encoding for performance optimization for a wide-variety of use cases , and are Unicode-correct to best support international languages and emoji. Tight-deterministic referencing counting is used to manage memory, keeping memory usage to a minimum without collection of garbage.

    Xcode

    Xcode 13 consists of powerful new team development features which allow collaboration with Xcode Cloud, Github and Bitbucket. Features include initiating, reviewing, commenting within the code, merging pull requests directly within Xcode and comparing different versions of code files.

    UIkit

    A UI kit is a collection of assets containing design elements such as UI components and styles, examples of UI components include navigation menus, widgets and input forms. UI components are elements that provide functionality and convey meanings to the users. UI kits typically consist of mock-ups for a solution in a particular field. UI kits can improve design workflow by:

  • Speeding up design processes: Ui kits allow the simplification of the task of finding a solution to a design problem. There are ready-to-use UI elements from the kit available instead of having to make new ones from scratch.
  • Achieving a consistent design: UI kits can become a foundation for the design system due to the design elements from a UI kit having a difficult consistency to achieve when combining multiple elements from different libraries.
  • Mastering design skills: Designers are presented with opportunities to inspect different components and learn how to build or improve similar components by themselves.

    Security

      Unsafe and Defective Networking
  • In a conventional mobile app, data will be transmitted between client and server. The app relies on the provider's network on the mobile device and also utilizes the internet whenever information is being sent. Cybercriminals can abuse mobile security breaches and steal confidential data and information from users. Due to the inaccuracy in the security mechanism, there is a threat of confidential information being leaked, such as session IDs, credentials and other sensitive data. Some mobile app developers utilize SSL/TLS certificates for authentication, however even the presence of a SSL/TLS does not guarantee the safety of the mobile application, therefore highly secured mechanisms must be implemented on the app to prevent the threat of a cyber breach.

      • Unreliable data storage
  • The risk of using unauthorized data storage can cause data breaches. A mobile app requires proper encryption algorithms to prevent hackers from having access to the mobile device.

    • UX/UI integration


      Poor UX/UI integration
  • The importance of a good UI and UX implementation is essential for mobile applications. A good user experience is crucial towards an application’s success. A user can traverse a web presence if a good interface is implemented. If an application is poorly designed and too complicated to use, it will cause dissatisfaction among users.
      • Developing an invaluable app for users
  • While developing the app, it is essential to remember the end-user’s needs the entire time. It is essential to conduct testing before launching the application as end users using the app are most knowledgeable about their requirements and needs.
      • Stuffing several functions in a single application
  • In the mobile app development industry, it is believed the more the functions, the greater the capabilities of the platform. However, having too many functions can cause the app to become too complex and complicated to use. The user might have trouble navigating throughout the app. It is essential to add the more significant features required by users and remove the redundant ones.

    • Cost


      Designing an App for Many Platforms at the same time
  • There are two primary app marketplaces, Google Play Store for android and App Store for IOS. There are roughly 2.2 million apps on the App Store and 2.6 million on Google Play store. For smaller businesses, there is a cost risk for the mobile app if launched on both platforms simultaneously. Two different development teams will be hired for each platform, and the cost of advertisements will also increase, thus making it difficult to launch on both platforms at the same time.
